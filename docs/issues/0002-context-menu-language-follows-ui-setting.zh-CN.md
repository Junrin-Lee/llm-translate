# 0002 — 右键菜单项未跟随浏览器语言

[English](./0002-context-menu-language-follows-ui-setting.md) · **简体中文**

- **报告时间:** 2026-07-15,截屏翻译冒烟期间
- **状态:** 按设计工作 —— 用户选择维持现状(无代码改动)
- **决策人:** 维护者,2026-07-15

## 现象

浏览器右键菜单整体为英文,扩展的二级菜单项("翻译此页"、"截屏翻译")却显示中文。

## 定位

不是 i18n 缺失。菜单标题走扩展统一的 `t()` 层,其语言由**「界面语言」**设置(`general.uiLang`,[src/i18n/index.ts](../../src/i18n/index.ts))决定;`auto` 时通过 `browser.i18n.getUILanguage()` 解析。截图中浏览器原生菜单出现 **"Translate to English"**——Chrome 只会提供翻译成浏览器自身 UI 语言的选项,证明浏览器 UI 语言是英文。菜单显示中文,意味着「界面语言」被显式设为中文;菜单忠实跟随了扩展级语言偏好,与 popup、卡片、设置页行为完全一致。

## 决策

维持现有语义:**右键菜单标题跟随扩展「界面语言」设置**,与其他所有界面一致。希望菜单跟浏览器语言的用户,把设置切回 Automatic 即可(整个扩展随浏览器语言)。

已评估但暂不采纳:按界面分层——浏览器原生表面(右键菜单)恒用 `getUILanguage()`、扩展内部界面跟设置。若混合语言菜单引发真实用户反馈可重启该方案,改动很小(给 `background.ts` 的菜单代码提供一个按浏览器语言取词的 `t` 变体)。
