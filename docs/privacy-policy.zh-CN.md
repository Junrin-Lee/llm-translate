# 隐私政策 — LLM Translate

[English](./privacy-policy.md) · **简体中文**

_最后更新:2026-07-14_

LLM Translate 是一个提供划词翻译、全文翻译与截屏翻译的浏览器扩展。它采用**自带密钥(BYOK)**模式:
由你提供自己的 OpenAI 兼容或 Anthropic 兼容 API Key,扩展直接与**你**配置的端点通信。
无账号、无我方运营的后端、无遥测。

## 概要

- 我们(开发者)**不运营任何服务器**,也**不从你这里接收任何数据**。
- 所有设置与 API Key 都**只保存在你的本机**,除下文所述外不会同步或传输到任何地方。
- 唯一的对外网络请求发往**你配置的翻译 API 端点**,且只携带你选择要翻译的文本。

## 扩展保存什么、存在哪里

所有数据都保存在你浏览器的本地扩展存储(`chrome.storage.local`)中,位于你自己的设备上。
远端服务器不保存任何东西。

- **服务商配置** —— 协议、Base URL、模型、可选参数,以及你填写的 **API Key**。密钥只存本机,
  **从不跨设备同步**。
- **偏好设置** —— 目标语言、界面语言、划词触发方式、整页展示模式,以及按站点的规则
  (自动翻译 / 禁用清单)。
- **自定义提示词** —— 你设置的任何自定义 Prompt 模板。
- **翻译缓存** —— 近期译文,保存在本地以避免重复翻译相同文本(划词结果存 session 存储,
  整页结果存 local 存储)。你可随时在设置页清空。

## 通过网络发送什么

当你翻译划词内容或整页时,扩展的 background service worker 会**直接向你配置的 API 端点**
发送以下内容:

- 你**选择要翻译的文本**;以及
- 你的 **API Key**,作为标准认证请求头(OpenAI 兼容端点用 `Authorization: Bearer …`,
  Anthropic 兼容端点用 `x-api-key: …`)。

这些请求**只**发往你填写的 Base URL。扩展不发起任何其它网络请求,不向我们或任何第三方
发送分析数据、使用统计、崩溃报告或任何可识别身份的信息。

由于端点由你选择,你翻译的文本将受**你所选服务商**(例如 OpenAI 或 Anthropic)的隐私政策
约束。请查阅该服务商的政策,了解他们如何处理你发送的内容。

## 截屏翻译

- 你框选的截取区域**仅**发送至截屏翻译解析到的 API 端点(其功能级覆盖,或全局默认
  Provider)——与划词、全文翻译发往同一类目的地。
- 截图内容不做任何持久化:不写入磁盘、不缓存;降级为扩展页流程时用于交接截图的内存
  暂存(`storage.session`)会在浏览器关闭时自动清除。
- 首次使用截屏翻译时会出现一次性提示,告知截图将发送至你配置的端点;确认后不会
  再次出现。

## 我们收集什么

**什么都不收集。** 开发者不收集、不接收、不存储、不出售、不分享你的任何数据。也没有服务器
来收集它。

## API Key 的处理

- 密钥只保存在本机,仅用于对你触发的请求进行认证。
- 密钥**从不**写入日志、错误信息或诊断输出。
- 导出设置时**默认不含 API Key**;仅在你显式勾选时才包含,并会有屏幕提示告知文件将以明文
  包含密钥。

## 权限

扩展只申请工作所需的最小权限;每项权限的理由见
[权限用途说明](../store-assets/justifications.md)。

## 本政策的变更

若本政策有变更,上方的"最后更新"日期会随之更新,新版本将发布在本仓库中。
